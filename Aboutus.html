The goal of Castle Siege is to use a basic design and game flow to attract lovers of war-based and turn-based tactical games. Our goals are to create a Unity application that can pull in combat and strategy-based demography. Make a breakthrough into the strategy game market. Castle Siege will utilize Unity as the game engine, and using Unity will be built as a “web-browser” game hosted on a cloud server, AWS EC2 as the chosen cloud instance provider, and will connect to a frontend and database using RabbitMQ as the message brokering system to connect all three containers into one seamless, asynchronous system. Boardgame-inspired turn-based game where two players take control of their armies and fight it out. The web game is hosted on a backend server that is connected to a Frontend and Database through the use of RabbitMQ. This web game will appeal to fans of turn-based and strategy-based war games. A detailed news and update every hour of the day for different sports. The updates from different leagues worldwide about transfers, managers getting sacked, or player injury or recovery. Data will be ‘scraped’ from the internet every hour based on keywords, with the latter being based on teams or players. This data will be preserved on a Database until the next hour with the subsequent ‘scraping’ and update. The messaging service will make a call to the Backend to perform a scraping and then another call to the database to do the updating. The Frontend will make a call to the Messaging Service to display the desired data and information for the logged-in user. .
